I'm an ML Research Engineer at Cambridge University Press & Assessment, building scalable ML systems, distributed pipelines, and governance frameworks. I like love AI. The possibility of an artificially sentient being drove me to coding in the first place. These days I work on models that can not only make decisions, but actually explain why – explainability, interpretability, the whole nine yards. Like any engineer at heart, I like problems. Mostly because I like solving them.
My toolkit spans the full ML stack – PyTorch, Hugging Face, and scikit-learn for model development; AWS, Docker, Terraform, and MLflow for keeping things alive in production; Kafka, Redis, Elasticsearch for when things need to go fast. Python is my first language, Java a close second. Whether it's fine-tuning LLMs, building drift detection pipelines, or shipping microservices – if it involves making machines smarter, I'm probably having a good time.
